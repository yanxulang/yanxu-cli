# 言令架构

## 分层

实现按职责分为五层：

1. 规格层验证名称、说明、顺序、重复、类型、默认值和命令树结构；
2. 解析层读取长短选项、位置参数和子命令，生成模式 v1 结果；
3. 补全层对已完成词元运行无副作用的轻量状态机；
4. 展示层从同一规格生成帮助、默认摘要和建议；
5. 错误层统一生成`YANLING_`代码并恢复结构化运行时详情。

命令对象只保存三个私有有序列：选项、参数、子命令。名称和说明公开只读。所有构建器成功后才追加定义，失败声明不会留下半配置项。

## 规格不变量

- 命令、选项、参数和子命令名称在各自作用域唯一；
- 短名唯一，内建帮助名称保留；
- 必需位置参数先于可选参数，余项位于末尾；
- 必需根位置参数和子命令不能共存；
- 子命令关系无环，深度不超过 32，总节点不超过 1024；
- 选项默认值在验证前复制，声明失败不改变原命令；
- `规格()`使用显式队列复制整棵树，不以宿主调用栈遍历深层结构。

子命令查找和输出保留声明顺序，帮助与候选因此可确定复现。

## 默认值所有权

默认容器复制使用显式待查、待填和深度队列，限制 64 层、每容器 4096 项和 1024 个容器节点。解析每次再次复制单值默认；多值选项总是新建空列。

默认值快照不解决循环容器，因为言序 JSON 与命令行默认值契约只接受可序列化树；循环或不可序列化 JSON 在声明阶段拒绝。标量文本仍可能很长，因此帮助只读取最多 32 字预览；非空容器只展示种类和顶层项数。

## 解析状态机

每个命令层维护：当前 argv 位、是否仍读选项、已见选项典和位置词元列。长选项先查真实名称，再考虑`no-`合成。短项逐字符扫描，取值项消费余文或下一个词元。

进入子命令时把剩余 argv 交给子命令层，父层先完成自己的位置和必需选项检查。结果保留父子层级，不把不同命令的同名选项压进一个扁平典。

解析参数总量在每层入口校验。子命令当前实现会再次扫描剩余参数的字符预算；深度最多 32，因此最坏仍有明确上界，常见单层命令只扫描一次。

## 补全状态机

补全把最后一项隔离为前缀，只扫描此前完成词元。它维护当前命令、完整路径、已见项、待值定义、选项模式、位置索引和首个无效项。进入子命令时重置当前层的已见和位置状态。

补全不调用完整解析器，原因有二：当前前缀通常不是合法最终值；交互适配器需要区分“待选项值”和“位置参数”，而不是只得到一次异常。它仍复用同一查找、配置和资源常量，并用测试对齐关键分支。

## 帮助生成

帮助从规格直接构造：先生成用法，再按选项、参数、子命令分节。可选位置参数与子命令并存时用`{<子命令>|位置参数}`表达互斥入口。帮助总长度构造后校验为不超过 1048576 字。

超长说明仍可能先进入行列再触发总量拒绝，但单说明已限制 16384 字，直接子命令和定义数量也有上限，不存在无界项数。默认值摘要在序列化前限制文本预览，并不深度序列化容器。

## 错误模型

内部`报错`统一构造`YANLING_<代码>：消息`。`错误详情`只把符合`YAN[A-Z0-9_]{1,61}`的前缀识别为稳定代码，其他运行时错误归为`YANLING_RUNTIME`，同时保留运行时源代码、类别、位置和踪迹。

直接解析传播错误；尝试解析只在外层捕获一次并返回固定形状。生成失败帮助使用独立安全边界，帮助错误不会替换最初解析错误。

## 权限与并发

库不执行 I/O，也没有全局可变状态。命令实例在声明阶段可变，完成后应按只读方式使用；解析、帮助、规格和补全只创建局部结果，不修改规格。言序同步执行流内可重复调用同一实例。

库不承诺对同一实例进行跨线程并发声明。需要共享时先完成所有构建，再由宿主按运行时并发模型协调只读调用。
