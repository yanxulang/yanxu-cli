# 从 0.1 迁移到 1.0

## 依赖

把范围与修订更新到稳定标签：

```toml
[依赖]
言令 = { git = "https://github.com/yanxulang/yanxu-cli.git", 修订 = "v1.0.0", 版 = "^1.0" }
```

最低言序仍为 1.1.6，没有新增第三方依赖或权限。

## 保留的入口

以下 0.1 公共入口保留名称和参数顺序：

- `命令(名称, 说明)`及只读`名称/说明`；
- `加开关`、`加选项`、`加默认选项`、`加多值选项`；
- `加参数`、`加余项`、`加子命令`；
- `解析`、`解析当前`、`尝试解析`、`帮助`。

有效 0.1 规格的长短项、组合短开关、默认、多值、位置和子命令语义保持。

## 结果变化

普通解析新增并固定：

```text
模式版本、命令路径、命令、帮助、帮助文、选项、参数、子命令、子结果
```

旧代码按已有键读取可继续工作。依赖完整典相等、键数或序列化快照的测试需要更新。子结果现在也含完整根路径。

`尝试解析`成功和失败都固定为六个键：

```text
模式版本、成功、结果、错误、错误详情、帮助文
```

失败新增结构化`错误详情`；迁移条件判断到`错误详情.代码`。成功分支的`帮助文`固定为空文。

## 更严格的规格

1.0 在声明阶段拒绝：

- 空、超长、带空白/控制符/等号或连字符开头的名称；
- 非单字符短名和内建帮助冲突；
- 重复选项、短名、参数和子命令；
- 必需参数位于可选参数后、余项后继续声明、重复余项；
- 必需根位置参数与子命令共存；
- 子命令环、过深或过宽树；
- 默认值类型不匹配、不可序列化或超过容器预算。

如果 0.1 应用依赖这些歧义声明，需要先修正规格，而不是捕获后继续。

## 解析变化

- 单值取值选项重复明确报`YANLING_PARSE_DUPLICATE`；
- 真正声明的`no-`长项优先于否定语法；
- argv 数量、单项和总字符超过上限会在解析前拒绝；
- 错误按规格、值、解析、配置和资源拆分为稳定代码；
- 未知长项默认附加有界拼写建议；
- `--help`生成的帮助包含完整子命令路径，并展示位置参数和元数据；
- 默认值与规格快照深度隔离，结果修改不影响后续调用。

## 新配置入口

默认行为尽量保持 0.1 直觉。需要严格或传统“首个位置参数结束选项”行为时使用：

```yanxu
命令.按配置解析（参数，{
    「允许穿插选项」：假，
    「拒绝重复开关」：真
}）；
```

`按配置解析当前`和`尝试按配置解析`提供对应入口。配置未知键不再被忽略。

## 新规格与补全

`规格()`返回不可回写内部状态的整树快照。`建议()`提供当前层前缀候选；`补全()`和`按配置补全()`提供带命令路径、上下文、待值元数据和无效项的模式 v1 结果。

补全输入最后一项是当前前缀。shell 适配器在用户刚键入空格时必须追加空文，否则最后一个完整选项仍会被当作正在补全的前缀。

## 迁移检查

1. 更新依赖范围和锁文件到`v1.0.0`。
2. 运行所有命令构建路径，修复新增规格错误。
3. 更新依赖完整解析结果典或帮助快照的测试。
4. 把错误消息判断迁移到`错误详情.代码`。
5. 确认重复单值选项和开关重复策略符合预期。
6. 检查是否把密码或令牌放进说明或默认值。
7. 若实现补全，按“最后元素为前缀”调整适配器并安全转义输出。
8. 同时运行树解释器与字节码 VM，对照嵌套、`--`和`no-`边界。
