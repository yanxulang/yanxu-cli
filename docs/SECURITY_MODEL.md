# 安全模型

## 资产与信任边界

言令处理两类输入：应用作者建立的命令规格，以及终端、脚本、编辑器或上级进程提供的 argv。两者都可能异常或巨大，但只有应用在解析后决定是否访问文件、网络、凭据或子进程。

库保护的资产是进程可用性、结果完整性和规格隔离。它不保护应用随后执行的业务操作，也不把解析成功等同于授权或净化。

## 主要威胁

### 资源耗尽

攻击者可能提交大量、超长或深层参数；应用错误也可能声明极宽命令树或巨大默认容器。实现设置公开硬上限，并以显式队列遍历命令树和默认容器，避免由合法上限触发宿主栈溢出。

帮助、补全和近似建议同样有输出与前缀限制。建议算法只判断一次插入、删除、替换或相邻换位，不构造无界编辑矩阵。

### 规格篡改

调用方可能在声明后修改默认来源、规格快照或某次解析结果。默认容器在声明和解析时复制，`规格()`深复制命令元数据，命令内部列保持私有。外部修改不回写后续调用。

命令对象本身仍是构建器；持有它的代码可继续调用`加…`方法。应用应在启动阶段完成声明，之后只共享解析接口。

### 歧义与绕过

真实`no-`选项先于合成否定，避免合法名称被重解释。`--`显式关闭当前层选项与子命令识别。必需参数顺序、余项终止性、重复名称和命令环在声明阶段拒绝。

解析配置严格验证键和值，不接受拼写错误后回退默认。补全和解析使用相同的穿插、否定、重复开关和帮助策略；交互提示不会故意建议配置禁止的语法。

### 信息暴露

帮助会展示说明和标量默认值预览；解析错误可能回显被拒绝的值；错误详情还保留运行时位置与踪迹。不得把密码、令牌或私密路径写进说明或默认值。向日志、遥测或远程客户端返回错误前，应由应用按数据分类脱敏。

长文本默认只预览 32 字，非空容器只显示种类和顶层项数。这是资源与意外泄露缓解，不是秘密管理功能；短秘密仍会显示。

### Shell 注入

言令不解析命令行字符串，也不执行结果。补全候选是普通文本，不保证对 Bash、Zsh、Fish、PowerShell 或其他环境已经转义。适配器必须使用目标 shell 的安全协议，不能把候选或解析值拼接到待执行字符串。

## 权限最小化

包清单申请的文件、网络、TCP/UDP 监听和环境权限均为空，进程与原生扩展为假。`解析当前`使用言序标准环境参数接口，但清单不授予任意环境变量读取。补全不访问文件系统，因此文件候选必须由拥有相应权限的外层应用显式提供。

## 非目标

- shell 词法、引用、变量和通配符安全；
- 路径规范化、文件访问控制和符号链接防护；
- 业务值范围、枚举、跨字段或鉴权验证；
- 子进程参数转义与退出码策略；
- 并发修改同一命令对象的同步；
- 敏感数据存储、擦除或加密。

这些责任属于 shell 适配器、应用验证层、权限系统或专门安全库。

## 安全测试

发布门禁覆盖名称控制字符、重复/顺序/环、32 层和 1024 节点命令树、256 项规格、4096 个 argv、单项和总字符、64 层默认容器、1024 默认节点、一兆帮助、128 字补全前缀，以及树/VM 一致性。边界测试使用真实最大值，不通过缩小用例掩盖实现问题。
