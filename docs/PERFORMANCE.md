# 性能与资源预算

## 硬上限

| 项目 | 上限 |
| --- | ---: |
| 名称 / 建议前缀 | 128 字 |
| 单项说明 | 16384 字 |
| 单命令选项 | 256 |
| 单命令位置参数 | 256 |
| 直接子命令 | 64 |
| 子命令深度 | 32 |
| 命令树节点 | 1024 |
| argv 项数 | 4096 |
| 单 argv 字符 | 65536 |
| argv 总字符 | 1048576 |
| 默认容器深度 | 64 |
| 单默认容器项 | 4096 |
| 默认容器节点 | 1024 |
| 单次候选 | 64 |
| 默认值帮助摘要 | 256 字 |
| 帮助总字符 | 1048576 |

这些是安全拒绝线，不是推荐 UI 尺寸。几百个选项或一兆字帮助虽然在契约内，仍不适合人类使用。

## 复杂度

- 声明选项、参数和子命令为当前作用域线性查重，最坏分别扫描 256、256、64 项；
- 子命令环和深度检查按被加入子树节点线性扫描，最多 1024 节点；
- `规格()`按命令节点、选项和参数总量线性复制；
- 单层解析按 argv 线性前进，长短选项查找最坏扫描当前 256 个定义；
- 嵌套解析在每层校验剩余 argv 字符预算，深度上限使额外扫描有界；
- 默认容器复制按容器节点和元素线性执行，使用显式队列；
- `建议`按声明顺序扫描，达到 64 个匹配后提前返回；
- 未知长项近似建议最多返回 3 项，每次比较与名称长度线性相关；
- 上下文补全扫描已完成 argv，并按同一线性选项查找和位置计数推进；
- 帮助按输出项和字符总量线性构造。

## 确定性负载

`benchmarks/解析与补全.yx`在默认一百万执行步预算内完成：

- 128 次根选项、组合短开关、取值、多值、子命令和余项完整解析；
- 128 次跨父选项值与子命令路径的上下文补全；
- 对解析布尔结果和候选总数做确定性断言。

发布门禁在言序 1.1.6 和当前稳定版中分别运行树解释器与字节码 VM，并比较最终值、输出和错误。它是防止算法或执行预算明显退化的回归负载，不是跨机器吞吐基准。

## 调优建议

1. 常用选项和子命令按用户认知排序；当前查找是线性的，极大规格也会影响每个词元。
2. 不要把动态业务对象全部声明成子命令或选项；补全业务值应在外层按需提供。
3. 只在生成文档或调试时调用整树`规格()`；解析不需要快照。
4. 命令实例在启动时构建一次并复用，避免每次请求重复验证和复制规格。
5. 不把超长说明或巨大默认值用作数据传输；帮助只适合人类可读元数据。
6. 在真实 argv 分布、运行时版本和目标后端上测量；不要把回归负载的时间当作服务级指标。
