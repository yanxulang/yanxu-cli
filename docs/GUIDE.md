# 言令使用指南

## 建立命令规格

`命令(名称, 说明)`创建一个可链式配置的命令。名称用于 argv 中的精确匹配；说明只用于帮助，不参与解析。

```yanxu
定 根 为 言令.命令（「包」，「管理依赖」）；

根.加开关（「verbose」，「v」，「详细输出」）
    .加选项（「registry」，「r」，「地址」，「文」，「注册表地址」，真）
    .加默认选项（「jobs」，「j」，「数量」，「数」，「并发数」，4）
    .加多值选项（「feature」，「f」，「能力」，「文」，「启用能力」）
    .加参数（「清单」，「清单路径」，假）
    .加余项（「包」，「包名」）；
```

声明入口：

| 入口 | 结果语义 |
| --- | --- |
| `加开关` | 逻辑值，默认假，可用`--no-name`否定 |
| `加选项` | 单个取值，可设为必需，无默认时未给出则为空 |
| `加默认选项` | 单个取值，未给出时返回声明默认值的深复制 |
| `加多值选项` | 每次出现追加转换值，默认空列 |
| `加参数` | 单个文本位置参数，可必需或可选 |
| `加余项` | 接收剩余全部位置参数的文本列 |
| `加子命令` | 注册命令对象，按声明顺序展示和补全 |

长名和值名必须为 1 至 128 字，不能以连字符开头，也不能含空白、控制符或等号。短名为空或单个有效字符。`help`、`帮助`和短名`h`由内建帮助保留。

## 取值类型

- `文`：保留 argv 原文；
- `数`：使用 JSON 数字语法解析，拒绝逻辑、空值和容器；
- `理`：接受`true/false`、`1/0`和`真/假`；
- `JSON`：恢复任意合法 JSON 值。

默认值必须与声明类型一致；JSON 默认值必须可序列化。容器默认值在声明、规格快照和每次解析之间隔离，调用方修改结果不会污染后续解析。

## 选项语法

```text
--name value
--name=value
-n value
-nvalue
-vq
--no-verbose
--
```

组合短项从左到右读取。遇到取值选项时，组合中的余文就是它的值；没有余文则消费下一个 argv。单值选项重复始终报错，多值选项可重复。开关默认允许重复且后值覆盖；开启`拒绝重复开关`后第二次出现报错。

真实声明的`--no-color`优先于否定语法。若同时声明`color`和`no-color`，`--no-color`命中后者；否定后者使用`--no-no-color`。

`--`只在当前命令层终止选项和子命令识别，此后文本按位置参数处理。

## 位置参数与子命令

必需位置参数必须先于可选参数，余项必须最后且只能有一个。命令一旦含必需根位置参数，就不能再加入子命令；反向声明也会被拒绝。

可选位置参数或余项可以和子命令共存，但二者是互斥入口：第一个位置词若精确匹配子命令便进入子命令，否则进入当前命令的位置参数。需要把同名文本当位置参数时先传`--`。

```yanxu
定 添加 为 言令.命令（「add」，「添加依赖」）
    .加开关（「dev」，「d」，「开发依赖」）
    .加参数（「包」，「包名」，真）；

根.加子命令（添加）；

定 结果：典 为 根.解析（【「-v」，「add」，「-d」，「http」】）；
言 结果【「子结果」】【「参数」】【「包」】；
```

父命令选项保存在父结果，子命令选项保存在`子结果`。每层`命令路径`都是从根到该层的新列。

## 解析配置

```yanxu
定 结果 为 根.按配置解析（参数，{
    「允许穿插选项」：假，
    「拒绝重复开关」：真
}）；
```

覆盖只接受[README 配置表](../README.md#解析配置)中的五个逻辑键。`默认解析配置()`每次返回新典；`解析配置(覆盖)`可用于提前验证并得到完整配置。

关闭`帮助短路`后，三个内建帮助词不再有特殊含义，并按未知选项处理。关闭`允许穿插选项`后，第一个位置参数会结束选项读取。解析和补全共用相同配置语义。

## 解析结果模式 v1

普通结果固定字段：

| 字段 | 含义 |
| --- | --- |
| `模式版本` | 固定为`结果模式版本`，当前为 1 |
| `命令路径` | 根到当前层的名称列 |
| `命令` | 当前命令名 |
| `帮助` | 是否由内建帮助短路 |
| `帮助文` | 帮助短路时的当前层帮助，否则空文 |
| `选项` | 当前层全部选项，包含默认与空值 |
| `参数` | 当前层位置参数典 |
| `子命令` | 命中的直接子命令名或空 |
| `子结果` | 下一层结果或空 |

帮助短路不要求必需项完整；它返回当时已经读取的选项和空参数典。父层进入子命令前仍会完成父层必需选项检查。

## 抛错与结构化尝试

`解析`适合希望直接传播失败的代码。`尝试解析`适合 CLI 入口，它把失败转换为固定结果：

```text
模式版本、成功、结果、错误、错误详情、帮助文
```

成功时`结果`非空，其余错误字段为空；失败时`结果`为空，`错误详情`含`代码/消息/源代码/类别/位置/踪迹`，并尽力生成根帮助。帮助自身超过上限时，失败结果的`帮助文`为空，不会掩盖原解析错误。

程序判断只读取`错误详情.代码`。完整代码目录见[ERRORS.md](ERRORS.md)。

## 帮助与规格快照

`帮助()`始终列出`-h`、`--help`和`--帮助`。自定义选项展示类型、必需性、重复性、否定形式和有界默认摘要；位置参数有独立章节。非空容器默认值只显示类型和项数，长文本只显示短预览。

`规格()`返回整个命令树的深复制，字段包括名称、说明、选项、参数和子命令。它适合生成外部文档、检查声明或构建适配器；修改快照不会改变命令。

## 补全

`建议(前缀)`只查看当前命令。`补全(已输入)`会解析已经完成的 argv、进入子命令并识别待值上下文。完整契约见[COMPLETION.md](COMPLETION.md)。

## 常见陷阱

- 把整行 shell 文本作为一个元素传入；言令需要分词后的 argv；
- 用错误消息判断分支；应使用稳定代码；
- 把秘密写成选项默认值或说明，随后被帮助文本展示；
- 误以为`尝试解析`会设置进程退出码或输出帮助；它只返回数据；
- 在余项后继续声明参数，或让必需位置参数与子命令共存；
- 修改`规格()`或解析结果，期待回写命令定义；所有暴露容器都按契约隔离；
- 把补全候选直接拼进 shell 命令；适配器仍须按目标 shell 安全转义。
