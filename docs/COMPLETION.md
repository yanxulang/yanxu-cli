# 上下文补全契约

## 输入模型

`补全(已输入)`和`按配置补全(已输入, 覆盖)`接收已经由 shell 或编辑器分词的`列<文>`。最后一个元素永远是当前前缀，之前元素视为已经完成：

```text
刚输入：工具 build --re
传入：【「build」，「--re」】

刚输入：工具 build --output <空格>
传入：【「build」，「--output」，「」】
```

空列表示根命令尚未输入任何内容。`【「--」】`仍是在补全长选项前缀；`【「--」，「」】`表示终止符已经完成。

## 输出模式 v1

每个分支都返回同一字段集合：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `模式版本` | 数 | 固定为`结果模式版本` |
| `命令路径` | 列<文> | 已识别的根和子命令路径 |
| `上下文` | 文 | `候选`、`选项值`、`位置参数`、`帮助`或`无效` |
| `前缀` | 文 | 输入最后一个元素，空列时为空文 |
| `候选` | 列<文> | 至多 64 个确定性候选 |
| `等待选项` | 文或空 | `选项值`时为长名称 |
| `值名` | 文或空 | 规格中的值占位名 |
| `值类型` | 文或空 | `文`、`数`、`理`或`JSON` |
| `无效项` | 文或空 | `无效`时首个不可继续的完整词元 |

所有列和典都是新建结果。调用方可以安全排序、过滤或追加展示信息；下一次补全不受影响。

## 上下文

### 候选

候选按声明顺序产生：每个选项依次是长名、可用的否定形式、短名，然后是子命令和三个内建帮助词。重复文本去重，达到 64 项即停止。

前缀必须从候选开头匹配。位置参数已经开始后，默认仍提供可穿插选项，但不再提供子命令。单值选项用过后不再建议；多值选项继续出现。严格重复开关配置会隐藏已用开关的长名、否定形式和短名。

### 选项值

一个完整取值选项尚未消费值时返回`选项值`。言令不猜测文件、枚举或业务值，候选为空；适配器使用`等待选项/值名/值类型`选择自己的提供器。短选项组和长选项使用同一状态，例如`-vo`在`o`需要值时也会进入该上下文。

### 位置参数

完成`--`、禁用穿插后出现位置参数，或当前非选项前缀没有子命令候选时返回`位置参数`。适配器可在这里提供文件、目录或业务候选。已经完成的位置词元会按规格计数；超过容量会进入`无效`，余项则可持续接收。

### 帮助

启用帮助短路时，已完成的`-h`、`--help`或`--帮助`使上下文变为`帮助`。适配器通常停止建议。关闭帮助短路后这些词按普通未知选项处理。

### 无效

已经完成的未知选项、非法否定、开关附值、禁止的重复或多余位置参数会进入`无效`。补全函数不抛解析错误，因为它服务于交互式适配器；`无效项`标明状态机无法继续的位置。资源和配置错误仍会抛稳定库错误。

## 与解析器的一致性

补全复用同一命令规格和配置，但只验证结构，不转换已完成的值。比如`--jobs 非数字`在补全时只是消费一个值；真正解析时会报`YANLING_VALUE_NUMBER`。这样补全不会在用户仍编辑内容时制造业务诊断。

子命令只在当前层尚未见位置参数时识别，取值选项的下一个词即使与子命令同名也只作为值消费，真实`no-`选项也和解析器一样优先于合成否定。

## 适配器责任

言令不提供 Bash、Zsh、Fish、PowerShell 或编辑器协议的字符串脚本。适配器负责：

1. 从目标环境安全取得 argv 与当前词；
2. 按上述规则构造`已输入`；
3. 在`选项值`或`位置参数`上下文查询文件或业务数据；
4. 按目标 shell 转义候选，不能直接执行或拼接未引用文本；
5. 对外部提供器设置自己的超时、数量和权限边界；
6. 把模式版本未知的结果视为不兼容，而不是猜测字段。

补全本身没有 I/O 权限、用户回调或后台状态，相同规格、配置和输入总是得到相同输出。
