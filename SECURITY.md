# 安全策略

## 支持版本

| 版本 | 安全更新 |
| --- | --- |

## 私密报告

请不要为尚未修复的安全问题创建公开 Issue。使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告功能，并提供：

- 受影响的言令和言序版本、操作系统与执行后端；
- 最小命令规格和 argv 复现；
- 可观察影响，例如拒绝服务、信息暴露或边界绕过；
- 已知缓解方式，以及是否涉及默认值、帮助或错误输出中的敏感数据。

维护者会确认报告、评估严重度并协调修复与公告。公开披露时间应在修复版本可用后协商。

## 安全边界

言令把命令规格和 argv 当作不可信数据进行结构与资源校验，但它不是 shell、沙箱、鉴权器或业务验证器：

- 不执行参数，不展开引号、变量、通配符或命令替换；
- 不读取文件、环境或网络，除非应用在解析后自行执行；
- 类型转换只支持文本、数、逻辑和 JSON；
- 帮助、错误和补全可能回显名称、说明或参数片段，应用不得把秘密写入规格说明或默认值；
- 解析成功只证明命令行符合声明，不证明路径安全、权限充足或业务值可信。

完整威胁模型见[docs/SECURITY_MODEL.md](docs/SECURITY_MODEL.md)。
