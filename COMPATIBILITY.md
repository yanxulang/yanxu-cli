# 兼容性

## 语言与平台

言令 1.0.0 的最低言序版本为 1.1.6。发布门禁同时使用 1.1.6 与当前稳定版验证源码格式、静态检查、树解释器、字节码 VM、包锁和 Release 构建，并在 Linux、macOS、Windows 上运行最低版本矩阵。

库是纯言序实现，没有原生扩展、系统命令或平台路径逻辑；公共解析结果不依赖操作系统。`解析当前`读取运行时已经提供的参数列，shell 分词差异不属于库的兼容层。

## 版本承诺

- `1.x`内保留现有公开名称、参数顺序和模式 v1 字段含义；
- 向结果典增加可忽略字段属于向后兼容变更，删除或改义需要新的主版本；
- 新增解析配置键会保留现有默认行为；未知键继续报错，便于发现拼写和版本不匹配；
- 稳定错误代码保留语义，错误消息可改进，不作为兼容接口；
- 明示的资源上限可在次版本提高，降低上限需要主版本或安全公告。

## 0.1 兼容面

1.0 保留`命令`、全部七个声明入口、`解析`、`解析当前`、`尝试解析`和`帮助`。有效 0.1 规格与参数继续解析；主要可观察变化是结果新增模式与路径字段、尝试结果固定键集合、帮助内容更完整，以及无效规格和失控输入使用更精确的错误代码。

详细变化见[迁移指南](docs/MIGRATION_1.0.md)。

## 后端一致性

测试、示例、性能负载和独立消费者都必须在树解释器与字节码 VM 中得到相同的返回值、输出和错误。`yanxu 兼容`报告是发布门禁的一部分，不把“两个后端都能运行”误当成“行为一致”。

## 依赖与权限

言令没有第三方依赖，锁图为空。包清单明确拒绝文件、网络、TCP/UDP 监听、环境、进程和原生扩展权限。未来新增依赖或权限必须在 README、变更记录和安全模型中说明，并按语义版本评估兼容影响。
