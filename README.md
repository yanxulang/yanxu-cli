# 言令

言令（`yanxu-cli`）1.0.0 是面向言序 1.1.6+ 的纯言序声明式命令行解析库。它把命令规格、解析策略、结构化结果、帮助生成和上下文补全放在同一套可检查模型中，适合构建单命令工具、带子命令的工程工具和 shell 适配层。

## 安装

```sh
yanbao add 言令 --git https://github.com/yanxulang/yanxu-cli.git --version '^1.0'
```

等价清单：

```toml
[依赖]
言令 = { git = "https://github.com/yanxulang/yanxu-cli.git", 修订 = "v1.0.0", 版 = "^1.0" }
```

言令没有第三方包依赖，也不申请文件、网络、监听、环境、进程或原生扩展权限。

## 五分钟示例

```yanxu
引「包:言令」为 言令；

定 程序 为 言令.命令（「构建器」，「构建言序项目」）；

程序.加开关（「verbose」，「v」，「输出详细日志」）
    .加默认选项（「jobs」，「j」，「数量」，「数」，「并行数量」，4）
    .加多值选项（「feature」，「f」，「能力」，「文」，「启用能力」）
    .加参数（「入口」，「入口文件」，真）
    .加余项（「文件」，「其余文件」）；

定 尝试：典 为 程序.尝试解析当前（）；

若 尝试【「成功」】 则
    定 结果：典 为 尝试【「结果」】；
    言 结果【「选项」】；
    言 结果【「参数」】；
否则
    言 尝试【「错误详情」】【「代码」】；
    言 尝试【「帮助文」】；
终
```

## 主要能力

- 长选项、短选项、组合短开关、`--name=value`和显式`--`终止；
- `--no-name`开关否定，并保证真实的`no-`名称优先；
- `文`、`数`、`理`、`JSON`取值转换，必需、默认和可重复选项；
- 必需、可选、余项位置参数与最多 32 层子命令；
- 可配置的帮助短路、穿插选项、否定、重复开关和未知项建议；
- 固定模式 v1 的解析、尝试和补全结果；
- 展示类型、必需性、默认摘要和完整命令路径的帮助文本；
- 面向 shell 适配器的有界、确定性上下文补全；
- 深复制的规格快照和默认值，调用方不能回写内部声明。

## 解析配置

`解析`和`解析当前`使用默认配置。`按配置解析`、`按配置解析当前`、`尝试按配置解析`和`按配置补全`接收局部覆盖：

| 键 | 默认 | 含义 |
| --- | --- | --- |
| `帮助短路` | 真 | `-h`、`--help`、`--帮助`直接返回帮助结果 |
| `允许穿插选项` | 真 | 位置参数后仍识别选项 |
| `允许开关否定` | 真 | 接受合成的`--no-name` |
| `拒绝重复开关` | 假 | 同一开关第二次出现时报错 |
| `未知选项建议` | 真 | 未知长选项附加至多三个近似候选 |

未知配置键或非逻辑值会被拒绝，不会静默忽略。

## 结果与错误

普通解析返回固定字段：`模式版本`、`命令路径`、`命令`、`帮助`、`帮助文`、`选项`、`参数`、`子命令`和`子结果`。嵌套命令保留逐层结果，并在每层写入完整路径。

`尝试解析`固定返回`模式版本/成功/结果/错误/错误详情/帮助文`六个键。失败时用`错误详情.代码`做程序判断；`错误`和消息仅供人阅读。库错误使用`YANLING_`前缀。

## 上下文补全

补全输入把最后一个元素视为正在输入的前缀；用户刚输入空格时应显式追加空文：

```yanxu
定 补全：典 为 程序.补全（【「build」，「--ta」】）；
言 补全【「命令路径」】；
言 补全【「上下文」】；
言 补全【「候选」】；
```

上下文为`候选`、`选项值`、`位置参数`、`帮助`或`无效`。补全只处理 argv 词元，不拆分或执行 shell 文本；引号、转义、路径和命令替换仍由调用它的 shell 适配器负责。

## 资源边界

| 项目 | 硬上限 |
| --- | ---: |
| 单命令选项 / 位置参数 | 256 / 256 |
| 直接子命令 / 子命令深度 | 64 / 32 |
| 命令树节点 | 1024 |
| 实参数 | 4096 |
| 单实参 / 实参总字符 | 65536 / 1048576 |
| 默认值容器深度 / 节点 | 64 / 1024 |
| 单次补全候选 | 64 |
| 帮助文本 | 1048576 字 |

这些是拒绝失控输入的安全上限，不是推荐的界面规模。面向人的命令通常应远小于它们。

## 已知限制

- 解析器接收已经分词的`列<文>`，不提供 shell 词法分析或进程退出码管理；
- 位置参数统一为文本，类型化位置参数应由应用在解析后校验；
- 补全不读取文件系统，也不执行用户回调；选项值候选由外层适配器提供；
- 同一命令不能同时声明必需根位置参数和子命令；可选位置参数与子命令并存时二者为互斥入口；
- 实例适用于同步声明和解析；构建完成后应按只读方式共享。
